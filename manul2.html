LogHound

Advanced Log Search Utility v8.0.0

Overview

LogHound is a powerful command-line utility designed to search through log files efficiently with advanced features and intuitive usage.

Compressed File Support

Search directly in .gz, .tar.gz, and .tgz files without manual extraction.

Smart Filtering

Filter by modification time or date patterns in filenames.

Colorized Output

Easily identify matches and errors with color-coded results.

Performance Optimized

Built-in timeout and file limits prevent system overload.

Comprehensive Logging

All search results are automatically logged with timestamps.

Easy Installation

Simple system-wide installation with automatic symlink creation.

Why Choose LogHound?

LogHound combines the power of grep with specialized features for log analysis, offering an intuitive interface that saves time and effort when troubleshooting systems or analyzing application logs.

Installation

System Requirements

  • Linux/Unix-based system (tested on Ubuntu, CentOS, Debian)
  • Bash shell (version 4.0 or higher)
  • Standard GNU utilities (grep, find, sed, tar, gzip)
  • Root access for installation (to create symlinks)

Installation Steps

To install LogHound system-wide with default configuration:

sudo ./loghound.sh --install

This command will:

  1. Create three search string files in the current directory:
    • search_string - General search patterns
    • ims_string - IMS-specific patterns
    • startUp_string - Startup-related patterns
  2. Create a symlink in /usr/local/bin/LogHound
  3. Create symlinks to search string files in /root/
Note

If the script isn't executable, run chmod +x loghound.sh first.

To completely remove LogHound from your system:

sudo LogHound --uninstall

This command will:

  1. Remove the /usr/local/bin/LogHound symlink
  2. Remove search string file symlinks from /root/
  3. Remove the ./log_find directory
Important

This will not remove your original search string files or the script itself, only the system-wide symlinks and log directories.

To upgrade to a newer version of LogHound:

# First uninstall the old version sudo LogHound --uninstall # Then install the new version sudo ./loghound.sh --install

Your search string files will be preserved during this process.

Pro Tip

Consider backing up your search string files before upgrading:

cp search_string search_string.backup cp ims_string ims_string.backup cp startUp_string startUp_string.backup

Post-Installation

After installation, you can run LogHound from any directory by typing:

LogHound [options]
Verification

Verify your installation by running:

LogHound -h

This should display the help message with all available options.

Command Options

LogHound provides numerous options to customize your search behavior:

Basic options
Filtering options
Output options
Admin options
Option Description Example
-s, --search-file Specify custom file containing search strings (default: search_string) -s ~/custom_strings.txt
-f, --file Search in a single specified file -f /var/log/nginx/error.log
-d, --directory Search all files in specified directory (recursive) -d /var/log/nginx/
-m, --multiple Search in multiple specified files -m app.log system.log
-t, --time Only process files modified in last X minutes -t 1440 (last 24 hours)
--date Process files with date pattern in names (yyyymmdd[hh[mm[ss]]) --date 202305 (May 2023)
-vvv, --verbose Print lines from target files in log file -vvv
--print Log matching lines with line numbers --print
--delete Delete all log_find directories/files --delete
--install Install script and create required files --install
--uninstall Remove all symlinks and log files --uninstall
-h, --help Show help message -h
Option Combinations

Most options can be combined for more powerful searches. For example:

LogHound -d /var/log/ -t 120 --print -vvv

This searches in /var/log/ for files modified in the last 2 hours, showing matching lines with numbers and verbose output.

Usage Examples

Here are practical examples demonstrating LogHound's capabilities:

Default search
LogHound

Searches in /data/storage/log for IMS*.log files using patterns from search_string.

Directory search
LogHound -d /var/log/nginx/

Searches all log files in the nginx directory (including compressed files).

Single file search
LogHound -f /var/log/syslog

Searches only in the system log file using default search patterns.

Custom search strings
LogHound -s ~/error_patterns.txt -d /opt/app/logs/

Uses custom patterns from error_patterns.txt to search in application logs.

Time-based search
LogHound -t 30

Searches only in files modified in the last 30 minutes.

Date pattern search
LogHound --date 20230615 -d /var/log/archive/

Searches files with "20230615" in their names in the archive directory.

Verbose search with line numbers
LogHound -vvv --print -d /var/log/

Shows matching lines with numbers and includes file contents in output.

Filtered verbose search
LogHound -vvv -t 1440 --date 202306 -d /var/log/

Verbose search in files modified in last 24 hours with "202306" in names.

Complex search scenario
LogHound -s critical_errors.txt --print -t 120 -d /opt/app/{logs,debug}/

Searches for critical errors in multiple directories, limited to recent files.

Troubleshooting server issues
LogHound -s server_errors.txt -t 30 -d /var/log/{nginx,mysql,apache2}/

Quickly checks multiple service logs for recent errors.

Debugging application crashes
LogHound -f /opt/app/debug.log --print -s crash_patterns.txt

Finds crash patterns in application debug log with line numbers.

Analyzing archived logs
LogHound --date 202301 -d /var/log/archive/ -s monthly_report.txt

Generates a monthly report by searching January 2023 archived logs.

Security audit
LogHound -vvv --print -s security_patterns.txt -d /var/log/{auth.log,secure}/

Comprehensive security audit with verbose output and line numbers.

Tips & Tricks

Search String File Format

The search string file supports various pattern types for flexible searching:

# Simple error matching
ERROR
# Wildcard patterns
*exception*
timeout*
# Basic regex patterns
failed.*login
connection refused
# Case insensitive matching (automatically applied)
critical
# Multiple variations
segfault|segmentation fault
# Combined patterns
error.*database
(warning|error).*disk

Performance Optimization

Time Filtering

Always use -t with large directories to limit search scope.

Targeted Directories

Be specific with directory paths to avoid scanning unnecessary files.

Compressed Files

Searching in .gz files is slower - filter them with --date when possible.

Output Interpretation

O
Match found
X
No match found
!
Error occurred
Log Rotation

When dealing with log rotation, combine --date with directory search:

LogHound -d /var/log/ --date $(date +%Y%m%d)

This searches logs from today only, automatically handling rotated logs with date suffixes.

Custom Configurations

Create multiple search string files for different purposes:

  • security_string - For security-related patterns
  • performance_string - For performance issues
  • startup_string - For initialization problems

Then use them as needed:

# Security audit LogHound -s security_string -d /var/log/ # Performance check LogHound -s performance_string -t 60
